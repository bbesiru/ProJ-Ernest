Shared Responsibility Model

The Shared Responsibility Model explains how security responsibilities are divided between Microsoft Azure and the customer.

Microsoft Azure Responsibilities

Physical security of data centers

Hardware maintenance

Network infrastructure

Platform and service availability

Underlying cloud infrastructure security

Customer Responsibilities

Managing user accounts and permissions

Protecting passwords and identities

Configuring the Storage Account securely

Protecting stored data

Monitoring and managing access to resources

How It Applies to the Deployed Storage Account

For the deployed Storage Account, Microsoft Azure is responsible for securing the physical data centers, networking, and cloud infrastructure that host the service. I, as the customer, am responsible for controlling who can access the Storage Account, configuring security settings, and protecting the data stored within it.
